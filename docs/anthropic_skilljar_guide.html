AI Fluency: Framework & Foundations
03 March 2026
Core Concept
What is AI Fluency?
In simple words

AI Fluency means knowing how to work with AI — not just using it blindly, but understanding when to use it, how to talk to it, how to judge its output, and how to use it responsibly.

Think of it like learning to drive. You don't need to know how an engine works internally — but you do need to know traffic rules, how to steer, when to brake, and how to be a responsible driver.

3 Ways people engage with AI
Automation

AI does the task for you. Example: "Write me a summary of this PDF." You give the task, Claude delivers.

Augmentation

You + AI collaborate. Example: You draft a blog post, Claude helps improve the tone, suggest headlines, and refine structure. You're partners.

Agency

You set Claude up to act independently. Example: You configure Claude with your company's style guide, brand voice, and templates — so every time you ask for a report, it follows your rules automatically.

Core Framework
The 4Ds — Your AI Collaboration Compass
Delegation — what to hand off

Deciding which tasks are better done by AI vs. yourself. Not everything needs AI, and not everything can be done without AI oversight.

Real-life example

Good delegation: "Claude, draft the first version of this project proposal." Bad delegation: "Claude, decide whether I should quit my job." (Too personal, too high-stakes for AI alone.)

Description — how to communicate

This is about prompting well. The clearer you describe what you want, how you want it done, and how Claude should behave — the better the result. Claude can't read your mind.

Try this prompt trick

"Before answering, please think through this problem carefully. Consider the different factors involved, potential constraints, and various approaches before recommending the best solution."

Discernment — evaluate what you get

AI outputs aren't always perfect. Discernment means asking: Is this accurate? Did Claude approach this the right way? Did it behave appropriately?

Real-life example

Claude gives you a list of statistics in a report. Discernment means verifying the numbers before sending that report to your boss — AI can hallucinate facts with confidence.

Diligence — be responsible

Disclose when AI helped. Verify before sharing. Be honest about what's AI-generated and take responsibility for what you put out into the world.

💡

Different contexts have different rules. Academic writing has strict AI policies. A personal email? Much more flexible. Know the rules of your context.

Practical Skills
6 Prompting Techniques That Actually Work
The six techniques
Give Context Show Examples Specify Constraints Break into Steps Ask it to Think First Define Role/Tone
Context example

❌ "Write an email."  →  ✅ "Write a polite follow-up email to a client who hasn't responded in 2 weeks about a proposal I sent. Keep it under 100 words, professional tone."

Secret weapon

Ask Claude to improve your own prompt! Try: "How could I rephrase my question to get a better answer from you?"

Claude 101
06 March 2026
Core Concept
Claude is More Than a Chatbot
Built to be: Helpful, Harmless, Honest

Unlike a search engine (just finds info) or a calculator (just computes), Claude reasons, understands nuance, asks for clarification, and collaborates. Think of it as a very capable intern who has read everything — but needs your direction.

The right mindset

Talk to Claude like a coworker — naturally, concisely, and conversationally. You don't need special magic words.

The 3-part prompt structure
Set the stage → Define the task → Specify rules

Set the stage: "I'm a marketing lead at an indie streaming startup preparing an investor pitch deck."

Define the task: "Research the current state of independent film streaming — trends, competitors, opportunities."

Specify rules: "Use web search, cite sources, structure as a professional report."

Feature Deep-Dive
Projects, Artifacts & Skills
Projects — your ongoing workspaces

A Project is like a shared notebook with Claude. You upload your reference materials (meeting notes, reports, templates), and Claude always has that context. No need to re-explain your world every conversation.

When to use a Project

You're managing a product launch for 3 months. Create a Project, upload your roadmap, brand guidelines, and meeting notes. Now Claude always knows the context when you ask "What should we prioritize this week?"

🧠

RAG Mode: When you upload lots of documents, Claude automatically uses "Retrieval Augmented Generation" — it doesn't forget your files, it smartly pulls the relevant bits. Up to 10x more capacity!

Artifacts — interactive outputs

Instead of plain text in chat, Artifacts are standalone, rendered outputs — a working website, a live chart, a downloadable document — visible in a panel beside your chat. Automatically created for complex, reusable content.

Real-life example

Ask Claude to "build a budget tracker in HTML" — it won't just show you code. It'll render a working, interactive budget tracker you can use right away.

Skills — expertise packages for Claude

Skills are pre-loaded instructions that make Claude really good at specific tasks. Anthropic provides built-in Skills for creating Excel, Word, PowerPoint, and PDF files. You can also create your own Custom Skills!

Custom Skill idea

Create a "Weekly Report" skill that always uses your company's template, brand colors, and writing style — so every report Claude writes for you looks like it came from your team.

⚡ Key distinction: Projects store knowledge. Skills perform tasks.

Power Feature
Connectors & Enterprise Search
What are Connectors?

Connectors link Claude to your actual tools — Slack, Gmail, SharePoint, Google Drive, and more. Claude can then search and synthesize information across all of them at once.

Real-life power example

"What happened in the business last week?" — Claude searches your Slack messages, emails, SharePoint docs, and Drive files simultaneously, then gives you a unified summary with citations.

Research Mode

Turns Claude from a conversational partner into a systematic investigator. It plans its research approach, conducts multiple searches, synthesizes findings, and cites everything. Think of it as having a research analyst on demand.

Claude Code in Action
16 March 2026
Core Concept
What Makes Claude Code Different?
Language Model vs. Coding Assistant

A plain language model can only read and write text — it can't touch your files or run commands. Claude Code adds "tool use" — it can actually read your files, write code, run commands, and make changes to your project.

Analogy

A language model is like a brilliant consultant who gives advice over the phone. Claude Code is that same consultant who's actually sitting at your desk, with access to your computer, your files, and your tools.

CLAUDE.md — your project's memory file

Run /init in a new project and Claude will analyze your codebase and create a CLAUDE.md file — a persistent summary of your project's architecture, commands, and style. It's included in every request automatically.

CLAUDE.md — shared with team CLAUDE.local.md — just you ~/.claude/CLAUDE.md — all projects
💡

Use the # command to add instructions to CLAUDE.md on the fly: type # Only add comments for complex logic and Claude merges it into your file.

Power Features
Planning Mode, Thinking Modes & Context Management
Planning Mode (Shift + Tab twice)

Before making changes, Claude reads more of your codebase, creates a detailed plan, shows you exactly what it intends to do, and waits for your approval. Great for big changes.

When to use

You're refactoring your authentication system across 10+ files. Enable Planning Mode first — Claude will map out exactly what it'll touch before doing anything.

Thinking Modes — depth on demand
Think Think more Think a lot Think longer Ultrathink

Each mode gives Claude more time and "mental space" to reason through hard problems before answering. Use for tricky bugs, complex algorithms, or architectural decisions.

Context Management
Commands

/compact — Summarizes conversation history, keeps key info, frees up space.
/clear — Fresh start, blank slate.
Double Escape — Rewind to an earlier point in the conversation.

Advanced
Custom Commands, MCP & GitHub Integration
Custom slash commands

Create your own /audit or /write_tests commands by putting markdown files in .claude/commands/. The filename becomes the command. They can accept arguments too.

/write_tests the use-auth.ts file in the hooks directory
MCP Servers — superpowers for Claude Code

MCP servers give Claude new abilities. For example, the Playwright server lets Claude control a real web browser — so it can see your website visually, not just the code, and make smarter styling decisions.

claude mcp add playwright npx @playwright/mcp@latest
GitHub Integration

Run /install-github-app and Claude becomes an automated team member on GitHub — it can respond to @claude mentions on issues/PRs, review pull requests, and run as a GitHub Action.

Intro to Agent Skills
21 March 2026
Core Concept
What Are Skills (in Claude Code)?
The core idea

Skills are reusable markdown files that teach Claude Code how to handle specific tasks — automatically, every time. Instead of explaining your PR review checklist over and over, write it once as a skill. Claude loads and applies it whenever you ask for a PR review.

Analogy

A Skill is like an SOP (Standard Operating Procedure) at a company. New employee? Read the SOP. Claude is the employee; your SKILL.md is the SOP.

CLAUDE.md vs. Skills vs. Slash Commands
The key difference

CLAUDE.md — loads into EVERY conversation (always on)
Skills — loads ON DEMAND when Claude detects the task
Slash commands — only when you explicitly type them

Skills are smarter than commands — they activate automatically based on what you're asking for.

Practical How-To
Creating Your Own Skills
Skill structure (it's just a markdown file!)
--- name: pr-description description: Writes pull request descriptions. Use when creating a PR or summarizing changes for a pull request. --- When writing a PR description: 1. Run `git diff main...HEAD` to see all changes 2. Write: ## What, ## Why, ## Changes

The description field is the most important part — it's how Claude decides when to use the skill. Write it like: "Use when [specific situation]."

Skill locations & priority
~/.claude/skills → personal, all projects .claude/skills → project-specific, shared

Priority order: Enterprise > Personal > Project > Plugins. If there's a conflict, higher priority wins.

Subagents — delegating with fresh context

A subagent is an isolated Claude instance you can delegate specific tasks to. It starts with a clean context (no baggage from your current conversation), which is great for focused, parallel work.

🤖

You can create a custom subagent (e.g., "frontend-security-reviewer") that automatically loads specific skills like accessibility-audit and performance-check every time it runs.

Intro to MCP
25 March 2026
Core Concept
What is MCP and Why Does it Matter?
The problem MCP solves

Say you're building an AI app where users can ask about their GitHub data. GitHub has hundreds of features — repos, PRs, issues, projects, webhooks… Without MCP, you'd have to write, test, and maintain all the integration code yourself. That's months of work.

Without MCP

You write code to call GitHub's API → define every tool schema → handle all edge cases → maintain it forever. 😩

With MCP

GitHub (or someone else) publishes an official MCP server. You connect to it. Done. Claude now has access to all GitHub tools. 🎉

MCP in plain English

MCP is a standardized connector between Claude and external tools/services. Like how USB-C is a standard that lets any device plug into any charger — MCP is the standard that lets Claude plug into any service.

⚡

MCP ≠ Tool Use. Tool use is how Claude calls tools. MCP is where those tools come from — pre-built, pre-tested, ready to use.

Technical Depth
Building with MCP: Tools, Resources & Prompts
Three things an MCP Server can provide
Tools — actions Claude can perform

Example: read_document(doc_id) or edit_document(doc_id, old, new). Claude calls these like functions.

Resources — data Claude can read

Like GET endpoints. A user types "@deposition.md" and your MCP server serves that file's content directly into the conversation.

Prompts — pre-built expert instructions

You craft and test a great prompt for reformatting documents. Bundle it in your MCP server. Now every user gets your expert-level prompt — they don't need to figure it out themselves.

Defining a tool (Python SDK)

It's surprisingly readable — just a function with a decorator:

@mcp.tool( name="read_doc_contents", description="Read the contents of a document." ) def read_document(doc_id: str): return docs[doc_id]

The MCP Python SDK handles all the protocol details. You just write normal Python functions.

Testing with the MCP Inspector

Run mcp dev mcp_server.py and get a live browser interface to test your tools interactively — no need to build a full app first. See real inputs, real outputs, debug in real time.
